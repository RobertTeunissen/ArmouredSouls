# Page PRDs

This folder contains Product Requirements Documents (PRDs) for specific pages and UI features in Armoured Souls.

## Purpose

Page PRDs define:
- User interface specifications
- Page functionality and behavior
- User interactions and flows
- Visual design requirements
- Data requirements for each page

## Contents

### Core Pages
- `PRD_LOGIN_PAGE.md` - Login and authentication page
- `PRD_DASHBOARD_PAGE.md` - Main user dashboard
- `PRD_ROBOTS_LIST_PAGE.md` - Robot listing and management
- `PRD_ROBOT_DETAIL_PAGE.md` - Individual robot details and upgrades
- `PRD_FACILITIES_PAGE.md` - Stable facilities management

### Battle & Competition
- `PRD_BATTLE_HISTORY_PAGE.md` - Battle history and replay viewer
- `PRD_BATTLE_RESULTS_PAGE.md` - Post-battle results display
- `PRD_BATTLE_STANCES_AND_YIELD.md` - Battle stance and yield threshold configuration
- `PRD_LEAGUE_STANDINGS.md` - League rankings and standings

### Economy & Progression
- `PRD_WEAPON_SHOP.md` - Weapon purchasing interface
- `PRD_INCOME_DASHBOARD.md` - Income tracking and analytics
- `PRD_HALL_OF_RECORDS.md` - Achievement and records display

### Administration
- `PRD_ADMIN_PAGE.md` - Admin panel and tools

## PRD Structure

Each page PRD typically includes:
1. Executive Summary
2. User Stories
3. Page Layout and Components
4. Functionality Requirements
5. Data Requirements
6. User Interactions
7. Edge Cases and Error Handling
8. Related Systems
9. Future Enhancements

## Usage

When implementing or modifying a page:
1. Read the corresponding PRD for requirements
2. Follow the design system guidelines in `docs/design_ux/`
3. Ensure all functionality requirements are met
4. Update the PRD if requirements change
