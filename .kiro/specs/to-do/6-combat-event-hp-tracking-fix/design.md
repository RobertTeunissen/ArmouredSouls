# Design Document: Combat Event HP Tracking Fix

## Overview

This design addresses the HP/Shield tracking inconsistency in combat events where `robot1HP`/`robot2HP` fields swap based on attacker/defender roles rather than maintaining consistent robot identity. The fix migrates all consumers to use the correct `robotHP`/`robotShield` maps that are already being generated by the combat simulator.

## Requirements Traceability

| Requirement | Design Section |
|-------------|----------------|
| 2.1 Consistent robot identity via robotHP/robotShield maps | Component 1: Combat Simulator (already correct) |
| 2.2 Message generator uses robotHP map | Component 2: Combat Message Generator |
| 2.3 Shield break detection uses robotShield map | Component 2: Combat Message Generator |
| 2.4 Admin portal uses robotHP/robotShield maps | Component 3: BattleDetailsModal |
| 2.5 N-robot compatible data model | Component 1 & 2 (maps scale to any robot count) |
| 3.1-3.5 Backward compatibility | Component 4: Legacy Fallback Helper |

### Component 2: Combat Message Generator Updates

#### 2.1 Helper Function: `getHPFromEvent()`

Create a helper to safely extract HP/Shield values from events with legacy fallback:

```typescript
private static getHPFromEvent(
  event: CombatEvent,
  robotName: string,
  context: { robot1Name: string; robot2Name: string }
): { hp: number | undefined; shield: number | undefined } {
  // Prefer robotHP/robotShield maps (correct source of truth)
  if (event.robotHP && robotName in event.robotHP) {
    return {
      hp: event.robotHP[robotName],
      shield: event.robotShield?.[robotName],
    };
  }
  
  // Legacy fallback for old battle data
  // Note: This may be incorrect for events where robot2 attacked
  if (robotName === context.robot1Name) {
    return { hp: event.robot1HP, shield: event.robot1Shield };
  }
  if (robotName === context.robot2Name) {
    return { hp: event.robot2HP, shield: event.robot2Shield };
  }
  
  return { hp: undefined, shield: undefined };
}
```

#### 2.2 Update `checkShieldBreak()`

Replace direct `event.robot1Shield`/`event.robot2Shield` access with the helper:

```typescript
private static checkShieldBreak(
  event: CombatEvent,
  context: { robot1Name: string; robot2Name: string },
  prevShields: Map<string, number>,  // Changed from robot1PrevShield, robot2PrevShield
  narrativeEvents: any[],
  ts: number
): void {
  if (!event.defender) return;
  
  const defenderName = event.defender;
  const prevShield = prevShields.get(defenderName) ?? 0;
  const { shield: currentShield } = this.getHPFromEvent(event, defenderName, context);
  
  if (prevShield > 0 && currentShield === 0) {
    narrativeEvents.push({
      timestamp: ts,
      type: 'shield_break',
      message: this.generateShieldBreak(defenderName),
    });
  }
}
```

#### 2.3 Update `checkDamageStatus()`

Replace direct `event.robot1HP`/`event.robot2HP` access with the helper:

```typescript
private static checkDamageStatus(
  event: CombatEvent,
  context: { robot1Name: string; robot2Name: string; robot1MaxHP: number; robot2MaxHP: number },
  thresholds: Map<string, Set<number>>,  // Changed from robot1Thresholds, robot2Thresholds
  narrativeEvents: any[],
  ts: number
): void {
  if (!event.defender) return;
  
  const defenderName = event.defender;
  const { hp: currentHP } = this.getHPFromEvent(event, defenderName, context);
  
  if (currentHP === undefined || currentHP <= 0) return;
  
  const maxHP = defenderName === context.robot1Name 
    ? context.robot1MaxHP 
    : context.robot2MaxHP;
  
  const robotThresholds = thresholds.get(defenderName) ?? new Set();
  const pct = (currentHP / maxHP) * 100;
  
  for (const t of [75, 50, 25]) {
    if (pct <= t && !robotThresholds.has(t)) {
      robotThresholds.add(t);
      thresholds.set(defenderName, robotThresholds);
      const msg = this.getDamageStatusMessage(defenderName, currentHP, maxHP);
      if (msg) {
        narrativeEvents.push({ timestamp: ts, type: 'damage_status', message: msg });
      }
      break;
    }
  }
}
```
